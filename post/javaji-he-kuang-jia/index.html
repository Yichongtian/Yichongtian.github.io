[Java]集合框架<一>

2023-03-05

集合框架的概述

  1. 集合、数组都是对多个数据进行存储操作的结构，简称Java容器。
    说明：此时的存储，主要指的是内存层面的存储，不涉及到持久化的存储。
  2. 1数组在存储多个数据方面的特点：
    • 一旦初始化以后，其长度就确定了。
    • 数组一旦定义好，其元素的类型也就确定了，就只能操作指定类型的数据了。比如：String[] arr; int[] arr1; Object[] arr2;
      2 数组在存储多个数据方面的缺点：
    • 一旦初始化，长度不可修改
    • 数组提供的方法有限，且效率不高。

Java集合可分为Collection和Map两种体系

  • Collection接口：单列数据，定义了存取一组对象的方法集合
    • List：元素有序、可重复的集合（ArrayList、LinkedList、Vector）
    • Set：元素无序、不可重复的集合（HashSet、LinkedHashSet、TreeSet）
  • Map接口：双列数据，保存具有映射关系"Key-value对"的集合（HashMap、LinkedHashMap、TreeMap、HashTable、Properties）

Collection接口和常用方法

Collection接口遍历元素的方式1：使用Iterator（迭代器）

基本介绍

  1. Iterator对象成为迭代器，主要用于遍历Collection集合中的元素
  2. 所有实现了Collection接口的集合类都有一个iterator()方法，用以返回一个实现了Iterator接口的对象，即可返回一个迭代器。当一个类实现了Iterable<T>接口时，它必须提供一个iterator()方法的实现。这个方法返回一个实现了Iterator<T>接口的类的实例，该类提供了具体的遍历实现，所以不会出现类型错误或者编译错误。
  3. Iterator仅用于遍历集合，本身并不存放对象

迭代器的执行原理

Iterator iterator = collection.iterator();
while (iterator.hasNext()){
    System.out.println(iterator.next());
}

iterator.next()的作用：1.指针下移 2.将下移后指向的元素返回
注意：在调用iterator.next()方法之前必须调用iterator.hasNext()方法。如果不调用且下一条记录无效，直接调用iterator.next()会抛出NoSuchElementExcrption异常
（冷知识：ctrl+j调出idea快捷键）

Collection接口遍历元素的方式2：使用for循环增强

增强for循环只能用于遍历集合或数组，底层仍然是迭代器（使用增强for时打断点会发现它依然会调用迭代器的hasNext()、next()方法）

基本语法

for (Object obj : collection) {
    System.out.println(obj);
}

ArrayList的注意事项：

  1. permit all elements，including null
  2. 由数组来实现数据存储
  3. ArrayList基本等同于Vector，除了ArrayList是线程不安全的（执行效率高），在多线程情况下，不建议使用ArrayList
    例如add()方法，并没有被synchronized修饰

扩容机制

  1. ArrayList中维护了一个Object类型的数组elementData。
    transient Object[] elementData;
    注意：transient： 使用 transient 关键字可以修饰成员变量，用于指示编译器不要将其序列化。当一个对象被序列化时，其所有非 transient 修饰的成员变量都会被写入到序列化流中，而 transient 修饰的成员变量则不会。通常情况下，对象的序列化和反序列化过程是通过 Java 的 ObjectInputStream 和 ObjectOutputStream 类来实现的。如果我们需要将一个对象序列化并写入磁盘或通过网络传输，但又不希望某些敏感数据（例如密码、密钥等）被序列化，那么我们可以将这些数据的成员变量加上 transient 关键字。
    需要注意的是，transient 关键字只能修饰成员变量，不能用于修饰方法或类。同时，当一个类实现了 Serializable 接口时，该类中的所有 transient 成员变量都不会被序列化。
  2. 一共有三个构造函数分别是ArrayList()、ArrayList(int)、ArrayList(Collection<? extend E>)
  3. 当创建ArrayList对象时，如果使用的是无参构造器，则初始elementData容量为0, 第1次添加，则扩容elementData为10,如需要再次扩容，则扩容elementData为1.5倍。
  4. 如果使用的是指定大小的构造器，则初始elementData容量为指定大小， 如果需要扩容，则直接扩容elementData为1.5倍。

add()方法需要进行扩容时的流程

public boolean add(E e) {
    //确保 size+1 有空闲空间
    ensureCapacityInternal(size + 1);  // Increments modCount!!
    elementData[size++] = e;
    return true;
}

private void ensureCapacityInternal(int minCapacity) {
    ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;

    // overflow-conscious code
    if (minCapacity - elementData.length > 0)
        grow(minCapacity);
}

//真正执行扩容的部分
private void grow(int minCapacity) {
    // overflow-conscious code
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1);
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    // minCapacity is usually close to size, so this is a win:
    elementData = Arrays.copyOf(elementData, newCapacity);
}

LinkedList底层结构和源码分析

  1. LinkedList底层实现了双向链表和双端队列特点
  2. 可以添加任意元素（元素可以重复），包括null
  3. 线程不安全，没有实现同步

LinkedList的底层操作机制

  1. 维护了一个双向链表
  2. 维护了两个属性first和last分别指向首节点和尾节点
  3. 每个节点（Node对象）里又维护了prev、next、item三个属性
  4. 所以LinkedList的添加/删除不是通过数组完成的，效率更高
private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}
